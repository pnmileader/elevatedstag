# The Elevated Stag CRM — Design System

## Who & How
Katie Fore, luxury men's custom clothier in San Antonio. Uses this CRM on an iPad held in one hand while standing at a fitting table. She glances down between measurements, taps with one thumb, looks up client history before appointments, and reviews revenue at end of day. This is her private business tool — no clients ever see it.

## Device
iPad portrait (768×1024) is primary. Must work on laptop too. Touch targets minimum 44px. Bottom tab bar for navigation (iOS-native pattern). No hover-dependent interactions — everything works with tap.

## Aesthetic Direction: "Leather-Bound Ledger"
The interface should feel like opening a beautifully crafted appointment book — warm paper tones, confident dark type, discreet gold details. NOT a tech dashboard. NOT a SaaS product. A bespoke tool for a bespoke business.

Inspired by: theelevatedstag.com (warm minimalism, sepia photography, serif typography, generous whitespace with restraint). But adapted for a working tool — more information density than the marketing site, while keeping the same warmth.

## Typography
- **Headings**: Libre Baskerville — warm editorial serif. Used for page titles, section names, and metric numbers.
- **Body**: Source Sans 3 — humanist sans-serif, highly readable at small sizes, warm character.
- **Scale**: 14px body (readable on iPad at arm's length), 11px labels, 20px page titles, 24px hero metrics.

## Spacing
- Base unit: 4px
- Content padding: 16px (matches iPad safe area feel)
- Section gaps: 12-16px
- Touch target minimum: 44px height
- List row height: 48px (comfortable one-thumb tap)

## Components

### Bottom Tab Bar
- Fixed bottom, 60px height + safe area padding
- 5 tabs: Dashboard, Clients, Orders, Calendar, Settings
- Active: gold icon + label. Inactive: ink-muted icon, no label.
- Background: charcoal with subtle top border

### Top Bar
- Fixed top, 52px height
- Left: "ES" monogram + search icon
- Right: context-dependent action button (+ New Client, etc.)
- Background: paper color, bottom rule

### Content Sections
- No card borders — separation through background color alternation and thin rules
- Section headers: 11px uppercase Source Sans 3, ink-secondary
- Content on paper background, alternate sections on surface-alt

### Tables / Lists
- 48px row height for tap targets
- Thin 1px rule dividers
- No alternating backgrounds in lists (use rules instead)
- Left-aligned everything

### Buttons
- Primary: charcoal bg, white text, 4px radius, 44px height
- Secondary: transparent, 1px rule border, 44px height
- Text: gold color, no border, inline

### Inputs
- 44px height (touch target)
- 1px rule border, 4px radius
- 14px font

### Status Indicators
- Small colored dot (8px) + text label
- Never large colored badges
